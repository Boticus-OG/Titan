## Design Philosophy

### Core Principles

| Principle | Application |
|-----------|-------------|
| **Knowledge in the World** | All state visible on-screen; no hidden modes |
| **Plate-Centric View** | Users track samples/plates, not infrastructure |
| **Progressive Disclosure** | Simple surface â†’ drill-down to complexity |
| **Actor Transparency** | Each actor exposes its state directly to UI |

## Responsive Behavior

### Desktop (1920x1080+)
- Full three-column layout
- All panels visible simultaneously

### Laptop (1366x768)
- Two-column layout
- Collapsible sidebars
- Tabbed secondary panels

### Tablet (1024x768)
- Single column with navigation drawer
- Touch-optimized controls
- Larger tap targets

---

## Accessibility

- WCAG 2.1 AA compliance
- Keyboard navigation for all interactions
- Screen reader support for status updates
- High contrast mode available
- No color-only status indicators (always paired with icons/text)
